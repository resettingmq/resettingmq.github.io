datatabels_utils 开发笔记 - 2. 请求的处理

2017年7月6日

datatabels_utils 开发笔记 - 整体架构
简单实例页面：客户-案件管理系统

DataTables是前端很使用的一个库，能够已列表呈现数据。它具有很高的定制性，能够从多种数据源读取数据，并能提供分页，排序，搜索等功能。

这里就试图结合Django的generic views, custom template tags等特性，编写简单的接口，使得DataTables的配置更加方便直观，提高代码的可重用性。

基本思想

DataTables用于控制列表的显示，主要用在某个model对应的ListView中。而每个model显示的项目，以及各项目的排序，搜索等配置也应该是不同的，所以在常规的使用中，每个model都对应这一份不同的DataTables配置对象。

在这里考虑以model为中心，model的属性中配置关联DataTables所需要的参数与信息。在ListView渲染模板的时候，将对应model的DataTable配置信息读出，生成HTML Table元素并生成DataTables javascript实例化配置代码，以实现每个DataTables实例差异化显示。

generic View的设计

DataTables工作在ajax模式下时，会对后端发起ajax请求以获取用于显示的数据。
所以最终要实现的DataTablesListView除了继承ListView之外，还需要继承自JsonResponse，以方便生成json响应。
为了简化URL设计，考虑在同一个view中同时实现对普通的HTTP请求响应以及对Datatables ajax的响应。

因此需要在DataTablesListView中重写继承自ListView的get()函数。该函数最终用来处理GET请求（其实在ListView中，POST等请求最终也是通过这个函数来处理的）。在get()函数中，通过判断请求是否来自于ajax，从而决定调用基类的哪个方法来生成最终的响应。

为了便于DataTables相关功能与其它generic view结合，实现一个名为DataTableMixin的类，这个类封装了一些同DataTable渲染以及数据生成相关的属性和方法。

DataTableMixin这个类继承自JsonResponseMixin以及JsonContextMixin。

DataTablesListView同时也继承自DataTableMixin。

JsonResponseMixin类实现了render_to_json_repsonse()方法，用于生成JsonResponse对象。

JsonContextMixin类中定义了get_json_context_data()方法，这个方法返回用于JsonResponse对象所需要的数据。

指定DataTables的列

为Model所关联的DataTables指定列，用来限定JSON返回数据，以及用于指定HTML Table中所要生成的列。
DataTablesMixin中定义了dt_column_fields属性。这个属性是一个字符串列表。

DataTablesMixin还定义了get_dt_query_fields()方法，从设置的dt_config类属性中获取所配置的显示列。

JSON数据的生成

在DataTableMixin中， 通过get_queryset()方法来获取数据源。

但是在这里并没有将get_queryset()返回的queryset对象序列化成JSON数据，而是通过queryset对象的values()方法，先生成一系列dict的queryset；再将这个queryset对象转换成list。

JSON数据中的键，由定义在DataTablesMixin中的dt_data_src属性定义。

同时，queryset的values()方法还支持指定获取的field，这样不仅能够避免不希望的数据返回到前端，还能够减少Web应用与数据库以及客户端之间的数据传输。

在DataTableMixin中重写get_json_context_data()方法，用于根据DataTables的配置信息以及HTTP请求信息，生成所需要的数据。

对DataTables的serverSide模式下请求的处理

在serverSide模式下，客户端发出的ajax请求包含一系列query args，对返回的结果集进行限定（例如分页，排序，搜索等等）。请求处理函数中，需要对这些query args进行处理。

所以，需要在get()函数中，将request.GET作为参数传入到get_json_context_data()中，在get_json_context_data()函数中集中对数据进行查询等处理。

同时，DataTables还要求后端返回一些额外数据，用于数据验证以及前端的显示（例如分页信息）。这些信息包括：

  • draw
  • recordsTotal
  • recordsFiltered
  • error（仅在由错误的情况下返回错误相关信息）
    因此，需要在后端代码中生成这些信息并且正确返回。

对于排序的处理：

  1. 首先从HTTP GET中获取排序的目标列以及排序的方向；
  2. 利用Djanngo order_by()修改queryset

同样，对于搜索的处理：

  1. 首先从HTTP GET中获取搜索的目标字符串；
  2. 因为只实现了全局搜索，所以在所有的列上根据搜索目标字符串构建filter()条件。

每个列上搜索条件的构建时通过委托到对应的DataTableColumn对象上实现的。通过调用DataTableColumn.get_filter_q_object()方法，返回一个Django Q对象。
