DataTables utils 开发笔记 - 4. 添加搜索、排序、分页等功能

2017年7月7日

简单实例页面：客户-案件管理系统

在前面的文章中提到了Django-DataTables utils关于搜索和排序的设计目标。

当DataTables运行在非serverSide模式下时，搜索、排序以及分页由DataTables在前端实现，后端代码不需要负责业务处理。但是当DataTables为serverSide方式时，所有的查询业务都发送至后端处理，会通过url query参数传递相关的搜索以及排序信息。后台获取这些信息，处理数据，再将数据转换成JSON返回。

为了应对这种情况，需要在后端模块中加入相应的逻辑。

排序功能（仅实现单列排序）

在单列排序的情况下，HTTP GET请求参数中order[0][column]指定用于排序的目标列；order[0][dir]指定排序的方向。

  1. 首先从HTTP GET中获取排序的目标列以及排序的方向；
  2. 利用Djanngo order_by()修改queryset

搜索功能

当DataTables工作于serverSide方式下时，HTTP GET请求参数中search[value]用于指定搜索的目标值，search[regex]用于指示是否采用正则表达式搜索。

  1. 首先从HTTP GET中获取搜索的目标字符串；
  2. 因为只实现了全局搜索，所以在所有的列上根据搜索目标字符串构建filter()条件。

每个列上搜索条件的构建时通过委托到对应的DataTableColumn对象上实现的。通过调用DataTableColumn.get_filter_q_object()方法，返回一个Django Q对象。

分页功能

在serverSide模式下，DataTables通过start以及length请求参数指定请求的数据范围。其中start指定开始的数据项的下标（从0开始计数），length指定请求数据的长度。

由于Django MultipleObjectsMixin中的分页功能基于django.core.paginator，而这个Django内建的paginator是通过数据页的编号来指定分页数据的。为了与DataTables的请求参数相匹配，并没有使用这个内建的paginator。

实现的逻辑很简单，在DataTablesMixin.get_json_context_data()这个方法中，对进过搜索、排序后的结果进行截取即可。
